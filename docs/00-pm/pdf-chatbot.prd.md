## 2. Problem Statement

### 2.1 핵심 문제

- 취업규칙, 근로계약서, 사내 노무 규정 등 **PDF 문서가 수십~수백 페이지**에 달함
- 필요한 정보를 찾으려면 문서를 열고 Ctrl+F로 검색하거나 처음부터 읽어야 함
- 인사·노무 담당자가 아닌 일반 근로자는 규정 해석 자체가 어려움
- 잘못된 해석으로 인한 노무 분쟁 리스크 존재

### 2.2 현재 해결 방법의 한계

| 현재 방법 | 한계 |
|-----------|------|
| PDF 직접 검색 | 문서 전체 구조를 알아야 하고 시간이 걸림 |
| 인사 담당자 문의 | 담당자 부재 시 즉각 응답 불가 |
| 범용 AI 챗봇 사용 | 회사 내부 규정을 모르므로 일반적인 답변만 제공, 할루시네이션 위험 |

## 5. Feature Specification

### 5.1 핵심 기능

#### F-01. PDF 텍스트 자동 추출
- 서버 시작 시 `docs/` 폴더의 모든 `.pdf` 파일을 읽어 텍스트 추출
- `pdf-parse` 라이브러리 사용
- 추출 실패 시 에러 로그 출력 및 해당 파일 건너뜀
- 텍스트는 메모리에 캐싱 (재시작 전까지 유지)

#### F-02. Claude API 연동 (서버사이드 전용)
- `POST /api/chat` 엔드포인트로 사용자 질문 수신
- 추출된 PDF 텍스트를 시스템 프롬프트에 포함
- `claude-haiku-4-5` 모델 호출
- **API 키는 환경 변수(`ANTHROPIC_API_KEY`)에서만 로드, 프론트엔드 미노출**

#### F-03. 할루시네이션 방지 시스템 프롬프트
```
시스템 프롬프트 핵심 규칙:
- 반드시 아래 PDF 문서 내용에만 근거하여 답변할 것
- PDF에 없는 내용은 "해당 내용은 문서에서 찾을 수 없습니다"라고 답변할 것
- 일반적인 노무 지식으로 추론하거나 보완하지 말 것
```

#### F-04. 말풍선 채팅 UI
- 사용자 메시지: 오른쪽 말풍선 (파란색 계열)
- 챗봇 답변: 왼쪽 말풍선 (회색 계열)
- 로딩 인디케이터 (답변 대기 중 표시)
- 입력창 + 전송 버튼 (Enter 키로도 전송 가능)
- 빈 입력 시 전송 차단

#### F-05. PDF 교체 가능 구조
- `docs/` 폴더의 파일만 교체하면 됨
- 재배포 없이 서버 재시작으로 반영 (Vercel 환경에서는 재배포 필요)
- 여러 PDF 동시 지원 (폴더 내 모든 .pdf 파일 병합)

### 5.2 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 보안 | API 키 서버사이드 전용, .env Git 제외 |
| 성능 | Claude API 응답 대기 시 로딩 표시로 UX 보완 |
| 호환성 | 모바일 브라우저 포함 반응형 UI |
| 배포 | Vercel 서버리스 함수로 배포 |

## 6. Technical Architecture

### 6.1 기술 스택

| Layer | Technology | Reason |
|-------|------------|--------|
| 서버 | Node.js + Express | Vercel 서버리스 호환, 간단한 API |
| PDF 파싱 | `pdf-parse` | Node.js 환경에서 간단한 API, 추가 의존성 없음 |
| AI API | `@anthropic-ai/sdk` (claude-haiku-4-5) | 공식 SDK, 할루시네이션 제어 용이 |
| 프론트엔드 | Vanilla HTML/CSS/JS | 빌드 도구 없이 배포 가능, 의존성 최소화 |
| 배포 | Vercel | 서버리스 함수 + 정적 파일 동시 호스팅 |

### 6.2 폴더 구조

```
my-pdf-chatbot/
├── api/
│   └── chat.js          # Vercel 서버리스 함수 (Claude API 호출)
├── docs/
│   └── *.pdf            # 노무 규정 PDF 파일 (교체 가능)
├── public/
│   ├── index.html       # 채팅 UI
│   ├── style.css        # 말풍선 스타일
│   └── app.js           # 프론트엔드 로직
├── lib/
│   └── pdfLoader.js     # PDF 텍스트 추출 모듈
├── vercel.json          # Vercel 라우팅 설정
├── package.json
├── .env                 # ANTHROPIC_API_KEY (Git 제외)
└── .gitignore
```

### 6.3 데이터 흐름

```
[사용자] → 질문 입력
    ↓ POST /api/chat { message }
[Vercel 서버리스 함수]
    ↓ docs/*.pdf 텍스트 추출
    ↓ 시스템 프롬프트 구성 (PDF 텍스트 + 할루시네이션 방지 규칙)
    ↓ Claude API 호출 (claude-haiku-4-5)
    ↓ { reply } 반환
[사용자] → 말풍선으로 답변 표시
```

## 7. Constraints & Risks

### 7.1 제약 사항

| 항목 | 내용 |
|------|------|
| API 비용 | Claude API 사용량에 따라 비용 발생 (Haiku 모델로 최소화) |
| 토큰 한도 | PDF 텍스트가 길면 컨텍스트 한도 초과 가능 → 최대 8,000자 절단 처리 |
| 세션 | 브라우저 새로고침 시 대화 내역 초기화 (영구 저장 미지원) |
| Vercel 무료 플랜 | 함수 실행 시간 10초 제한 |

### 7.2 위험 요소

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| PDF 텍스트 추출 실패 (스캔본 등) | High | Medium | 텍스트 레이어 없는 PDF 감지 후 안내 메시지 |
| Claude API 토큰 초과 | High | Medium | 텍스트 최대 8,000자로 슬라이싱 |
| Vercel 함수 콜드 스타트 지연 | Low | High | 로딩 인디케이터로 UX 보완 |
| API 키 노출 | Critical | Low | .env + .gitignore + 서버사이드 전용 처리 |

## 8. Success Criteria (Definition of Done)

### 8.1 필수 완료 조건

- [ ] `docs/` 폴더에 PDF 추가 후 질문 → PDF 기반 답변 정상 동작
- [ ] PDF에 없는 내용 질문 시 "문서에서 찾을 수 없습니다" 응답
- [ ] API 키가 프론트엔드 코드(`public/`)에 노출되지 않음
- [ ] Vercel 배포 후 URL로 접속하여 챗봇 정상 동작 확인
- [ ] `.env` 파일이 Git 히스토리에 포함되지 않음

### 8.2 품질 기준

- [ ] 말풍선 UI가 모바일에서도 깨지지 않음
- [ ] 빈 메시지 전송 시 전송 차단
- [ ] 답변 대기 중 로딩 표시
- [ ] 여러 PDF 파일이 있을 때 모두 통합하여 답변

## 10. 사용자 가이드 — PDF 교체 방법

### 10.1 로컬 환경

```bash
# 1. 기존 PDF 삭제 (선택)
rm docs/기존파일.pdf

# 2. 새 PDF 복사
cp /path/to/새규정집.pdf docs/

# 3. 서버 재시작 (변경 사항 반영)
node server.js
```

> ⚠️ 서버를 재시작해야 새 PDF가 메모리에 로드됩니다.

### 10.2 Vercel 배포 환경

```bash
# 1. docs/ 폴더에 새 PDF 파일 복사
cp /path/to/새규정집.pdf docs/

# 2. Git 커밋 & 푸시 (자동 재배포 트리거)
git add docs/새규정집.pdf
git commit -m "docs: 노무 규정집 업데이트"
git push

# → Vercel이 자동으로 재배포하며 새 PDF가 반영됩니다
```

### 10.3 주의사항

| 항목 | 내용 |
|------|------|
| 파일 형식 | `.pdf` 확장자만 지원 (스캔 이미지 PDF는 텍스트 추출 불가) |
| 파일 수 | `docs/` 폴더 내 모든 .pdf 파일이 병합되어 사용됨 |
| 파일 크기 | 텍스트 추출 후 8,000자 초과 시 자동 절단됨 |
| 파일명 | 한글 파일명 사용 가능, 공백은 언더스코어로 대체 권장 |

### 10.4 스캔본 PDF 처리

텍스트 레이어가 없는 스캔본 PDF는 자동 추출이 불가합니다.

- **확인 방법**: PDF를 열고 텍스트 드래그·복사가 되는지 확인
- **해결 방법**: Adobe Acrobat 또는 온라인 OCR 서비스로 텍스트 레이어 추가 후 저장
